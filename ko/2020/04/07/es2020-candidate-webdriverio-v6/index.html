2020-04-07: ES2020 Candidate, WebdriverIO v6, 브라우저의 릴리스 정보를 모니터링하는 방법

2020年04月07日 Translator: U-Yeong Ju Edit on GitHub See Revisions
  • Tweet

JSer.info #482 - ECMAScript 2020의 릴리스 후보가 공개됐습니다.

  • Release ES2020 Candidate · tc39/ecma262

ES2020 Candidate는 TC39에서 승인된 형태로, 2020년 6월에 있을 Ecma의 General Assembly에서 승인되면 정식으로 ECMAScript 2020으로서 릴리스됩니다.

또, 이번 3월 부터 4월에 걸쳐 행해진 TC39 미팅에서 import.meta가 Stage 4로 변경됐습니다.

  • ECMAScript proposal updates @ 2020-04 | ECMAScript Daily

이로인해 import.meta도 아슬아슬하게 ES2020에 포함되게 됐습니다.
ES2020에 포함되는 Proposal은 아래 문서에 정리돼 있습니다.

  • proposals/finished-proposals.md at master · tc39/proposals
  • ECMAScript 2020: the final feature set

WebdriverIO v6가 릴리스 됐습니다.

  • WebdriverIO v6 Released · WebdriverIO

WebdriverIO v6에서는 devtools가 기본적으로 활성화 됩니다.
WebdriverIO는 Webdriver API를 사용하며, 현재의 Webdriver API에서는 접속한 브라우저의 개발자 도구를 사용할 수 없습니다.
때문에 devtools에서는 Puppeteer를 통합해 이 문제를 회피합니다.

  • Chrome DevTools as Automation Protocol · WebdriverIO

이와 관련해 Webdriver API 쪽에도 MSEdge Team에 의한 양방향 통신 제안이 공개돼 있습니다.

  • MSEdgeExplainers/WebDriverRPC at master · MicrosoftEdge/MSEdgeExplainers

이 외에 비권장된 request 대신 got를 사용한 성능 개선이 이뤄졌으며 타입스크립트 지원이 추가됐습니다.

JSer.info에서도 브라우저 릴리스 정보를 모니터링하고 있으며 이와 관련해 브라우저 릴리스 정보 사이트를 모아봤습니다.

  • 브라우저 / 보안 / 정보 보호 관련 1차 정보에 가까운 2차 정보의 정보원을 정리한 문서(일본어)

실제로는 좀 더 많은 리소스를 살펴보고 있지만 이를 간추려 정보원에 가까운 곳을 정리했습니다.

공개한 글과 함께 브라우저 공식 블로그나 이슈, SNS 등을 정리한 browser-resources도 갱신하고 있으니 살펴봐주시길 바랍니다.

  • azu/browser-resources: A Collection of official Resources/Status/Issues for browsers.

이 외로 브라우저 관련 RSS 등을 정리한 다음 사이트가 도움이 될 것 같습니다.

  • Web Browser Eng
  • PaulKinlan/topicdeck: An NPM module that hosts and aggregates collections of RSS feeds.
    • Chrome 팀의 Paul Kinlan이 관리하고 있다
