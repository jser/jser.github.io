JSer.info #416 - WebDriver를 사용하는 테스트 프레임워크 WebdriverIO v5가 릴리스됐습니다.

  • WebdriverIO v5 Released · WebdriverIO
  • webdriverio/CHANGELOG.md at master · webdriverio/webdriverio

패키지 구조와 API와 관련한 부분을 변경했습니다.

기본 패키지를 @wdio로 이동했고 원래 포함돼 있던 CLI를 @wdio/cli로 분리했습니다.

API 관련해서는 커맨드에 직접 selector를 전달 하지 못하도록 변경했고 메서드 체인이 아닌 async/await 기반으로 변경했습니다.

// v4에서는 셀렉터를 전달함
browser.click('#elem');
// v5에서는 element를 가져온 후 조작함
const elem = $('#elem');
elem.click()

또한 isVisible을 isDisplayed으로 이름을 변경했으며 이 외에도 많은 커맨드의 이름을 변경했습니다.
자세한 내용은 CHANGELOG.md에 정리돼 있으니 참고하시길 바랍니다.

모듈 번들러 Rollup v1.0.0가 릴리스됐습니다.

  • Release v1.0.0 · rollup/rollup

특정 옵션을 비권장하고 옵션의 이름을 변경하는 등 비호환 변경 사항을 포함한 릴리스입니다.

bundle.generate와 bundle.write 반환 값의 포맷을 변경했으며 사용중인 파서를 acorn@6로 업데이트 했습니다.

  • 1.0 changes by guybedford · Pull Request #2293 · rollup/rollup

Comlink + Rust로 언어와 스레드의 울타리를 넘은 WebAssembly 개발 - Qiita(일본어)에는 Rust를 사용한 WebAssembly 개발에 대해 Zip 압축 해제 앱을 예로 소개하고 있습니다.

Rust나 wasm-pack을 사용한 개발 환경 설정부터 wasm-bindgen로 Rust와 JavaScript 바인딩 작성, webpack으로 가져오기, 브라우저에서의 실행까지 작성돼 있습니다.
또, 메인스레드의 비용을 줄이기 위해 WebAssembly 처리를 WebWorker 상에서 실행하고, 메시지를 주고 받는데 Comlink를 사용하는 방법에 대해서도 소개합니다.
